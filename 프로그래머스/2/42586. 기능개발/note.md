## 1. 🔍 어떻게 시도했는가? (How)

### ❌ [FAIL / SO-SO] 최초 접근 로직
* **🤔 생각한 흐름:**
    * 줄줄이 소세지처럼 나열되어 있는데, 하나씩 값을 비교해가면서 빠지는 모양이 `Queue`가 연상됐다.
    * progress와 speed를 통해 잔여 기간 remaining을 계산할 수 있겠다.
    * 가장 앞선 remaining보다 큰 remaining이 나올 때까지 순회하고, 순회 횟수를 리스트에 추가하면 되겠다.
    * 모든 remaining이 없어질 때까지 위 조건에 맞춰 dequeue를 하면 될 것이다.
    * 배열을 Queue 형태로 복사할 수 있으면 좋겠다. 리스트를 배열로 복사할 수 있으면 좋겠다.

**Pseudo code**
```java
// progress와 speed를 통해 remaining 계산해서 Queue 만들기
// 변수 초기화 하기
  // int max = 0
  // int count = 1
  // List list = new List
// while (remaining.size != 0)
  //  if (max < peek)
    // max = peek
    // list.add(count)
    // count = 1
// 리스트를 배열로 바꿔 반환하기
```

* **⚠️ 왜 비효율적/실패했는가? (Self-Feedback):**
    * 끔찍한 실수가 반복돼서 시간도 오래 걸리고 코드도 지저분하다. 굳이 Queue 자료구조까지 필요 없었다.


### ✅ [SUCCESS] 정답/모범답안 로직
* **✨ 수정된 흐름:**
    * 첫 번째 요소를 계산해서 max에 할당한다.
    * 두 번째 요소부터 순회한다.
    * max보다 작거나 같을 경우 count++
    * max보다 클 경우 count를 list에 추가하고 count를 0으로 초기화
    * 마지막 남은 요소는 비교대상이 없으니 마지막 순회에는 count를 무조건 list에 추가한다.
    * list를 array로 전환하기

### 🛠️ 내 코드에서 놓친 디테일 (Syntax / Edge Case)
* **🚨 엣지 케이스:**
    * poll을 조건에 맞는 경우에만 했기 때문에 로직이 지저분해졌다. 훨씬 더 간결하게 풀 수 있었다.

## 2. 🎯 한 줄 본질 (What)
* 이 문제는 결국 `반복문`를 활용해서 `두 값을 비교`하는 문제다.

---

## 3. 💥 아차! 포인트 (Why)
* 내가 풀이 과정에서 막히거나 오판했던 점:
    * *Queue가 연상된다는 이유만으로 더 간단한 배열을 제치고 무작정 풀이를 시작했다. Queue를 사용해야 하는 근거가 없었다.*
    * *Queue의 poll을 조건에 부합할 경우에만 호출했기 때문에, 요소마다 값을 비교하는 패턴이 성립하지 않았다. 천동설을 뒷받침하기 위해 해괴한 논리가 필요했던 것처럼, 부자연스러운 패턴으로 문제를 풀고자 해서 while 반복문 속 로직이 괴상해졌다.*

---

## 4. ⚡ 다음에 보면? (Trigger)
* 자료구조, 알고리즘을 선택할 때는 단순히 연상된다는 이유가 아니라, 명확한 근거를 찾도록 하자.
* 반복문에서 `요소마다 비교하는 패턴`에서 벗어난 게 아닌지 점검해보자.
