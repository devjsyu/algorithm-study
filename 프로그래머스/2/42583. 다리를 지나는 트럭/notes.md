## 🧠 접근 흐름 (사고 과정 기록)
1. 처음 접근: 전형적인 Queue 형태라는 것은 일찌감치 포착했다. 배열에서 요소를 이동시키는 걸 떠올렸다. 그러다가 배열 요소를 반복문으로 옮기는 건 부적절하다고 생각해서 단념했다. 그리고 waiting-time을 반복문을 돌지 않고 직접 연산하는 방법을 떠올렸다. 하지만 이건 각 트럭 요소마다 시간을 관리해야 되서 커스텀 클래스를 만들어야 할 것 같았다. 그리고 queue에서 길이만큼 1씩 add, poll하는 것도 생각했지만 잘못된 발상이라는 걸 뒤늦게 깨달았다.
2. 막힌 지점: waiting-time을 어떻게 구해야 할지 헤맸다!
3. 수정한 방향: (답지 확인) 무게가 0인 요소를 추가하는 동시에 시간을 1씩 증가시키면, 자연스럽게 큐(배열) 속에서 요소가 이동하는 효과도 있고 waiting-time도 누적 합산할 수도 있다!

---

## 💡 핵심 아이디어 (진짜 중요)
👉 이 문제의 본질 = add 하는 요소의 값을 0 지정하면 Queue 자료구조를 활용하면서 waiting-time을 자연스럽게 구할 수 있다는 점이다.

- 핵심 패턴: condition and while-loop, inserting 0 value while incrementing time variable
- 사용한 자료구조: Queue
- 시간복잡도: O(N * M)
