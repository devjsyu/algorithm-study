# BOJ 10813 공 바꾸기

## 문제 요약
입력 받은 정보와 반복문을 활용하여 배열 요소를 교환하기

## 핵심 아이디어
변수에 할당된 값을 서로 교환하기 위해 임의의 변수에 값을 임시로 저장하여 교환에 활용한다.

## 시간복잡도
O(N)
총 n번 + m번 반복

## 내가 처음 사용한 접근
split 메서드를 사용

## 실수
- split를 사용해서 String[] 생성 비용, 정규식 기반 파싱 비용, 가비지컬렉션 부담이 불필요하게 생겼다.
- 불필요한 주석이 많았다.
- 불필요하게 변수 선언을 실제 사용되는 반복문 밖에 놓아 부자연스러웠다.

## 해결 방법
- 알고리즘 문제에 한해서는 입력 크기가 크다면 StringTokenizer를 사용하는 것이 바람직하다.

## 배운 점
- FastReader라는 임의의 클래스를 정의해서 빠르게 입력 받을 수 있다는 것을 알게 되었다.
- 간단하게 풀 수 있다면 굳이 큰 성능상의 이점도 없는데 추상화를 하지 말자.
