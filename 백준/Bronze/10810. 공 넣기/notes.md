# BOJ 10810 - 공 넣기

(예시 작성)

## 문제 요약  
N개의
바구니가 있고 M번의 연산이 주어진다. 각 연산은 (i, j, k) 형태이며
i번부터 j번 바구니까지 k번 공을 넣는다. 기존 공이 있으면 덮어쓴다.

## 핵심 아이디어  
이 문제는 단순한 구현 문제이다.

각 연산마다 다음을 수행한다. 
1. i부터 j까지 반복 
2. 해당 바구니 값을 k로 overwrite

## 시간복잡도 
O(N × M)

문제 제한 N ≤ 100 M ≤ 100

최악의 경우 100 × 100 = 10,000 연산

따라서 단순 반복문 구현이 가장 직관적인 해결 방법이다.

 
## 내가 처음 사용한 접근
 
- 크기 N 배열 생성 
- 모든 원소 기본값 0 
- 각 연산마다 - start ~ end 반복
- array[index] = ball

입력은 1-based index이므로 배열에서는 i - 1 처리가 필요하다.

 
## 실수 

1.  Off-by-one 오류

처음에는 다음처럼 작성했다.

`for (int i = first; i <= last; i++)`

하지만 배열은 0-based index이므로 범위 오류가 발생했다.

올바른 코드는 `for (int i = first - 1; i <= last - 1; i++)`이다.

2.  출력 형식 실수

마지막 원소 뒤에도 공백을 출력했다.

## 해결 방법 
- 마지막 원소가 아닐 때만 공백 추가 또는 StringBuilder 사용

 
## 배운 점 

1.  입력 index가 1-based인지 항상 확인해야 한다.

백준 문제에서 매우 흔한 패턴이다. 입력이 1 ≤ i ≤ j ≤ N 형태라면 대부분 배열에서 index - 1 처리가 필요하다.

2.  구현 문제에서도 변수명이 중요하다.

parametersParsed[0] 보다는 `start` `end` `ball`처럼 의미 있는 이름을 사용하는 것이 좋다.

3.  배열 출력에서는 StringBuilder 패턴이 자주 사용된다.
