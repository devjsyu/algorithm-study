## 1. 🔍 어떻게 시도했는가? (How)

### ❌ [FAIL / SO-SO] 최초 접근 로직
* **🤔 생각한 흐름:**
    * early exit
        * m과 n이 각각 0이면 그대로 nums2, nums1이다.
    * MergeSort의 구현 로직을 차용한다.
        * 자료구조 두 개에서 요소를 뽑아서 비교하고 최종 결과 자료구조에 집어넣는 과정 반복
        * 기존 자료구조 두 개 중 일부가 요소가 소진될 경우 따로 나머지를 뽑는 과정 반복

* **⚠️ 왜 비효율적/실패했는가? (Self-Feedback):**
    * 공간복잡도 관점에서 비효율적
        * In-place 정렬 구현 실패 : 임의의 배열 만들고 nums1 배열 원소 교체하는 방식 대신 nums1의 원소를 재배치하는 방식으로 구현했어야 했다.
    * 비효율적인 `Two pointers` 활용
        * `Two pointers` 개념을 뚜렷히 파악하지 못했기 때문에 더 효율적인 방향으로 활용하지 못했다.
        * index가 0부터 시작하는 게 아니라, 역방향으로 시작했더라면, nums1 배열 In-place 정렬을 자연스럽게 구현할 수 있었다.
    * 가장 효율적인 풀이 방법을 생각해낸 게 아니라, 문제 설명을 보고 가장 먼저 연상된 풀이 `MergeSort 구현 로직 차용`으로 풀었기 때문에 가장 효율적인 풀이가 아닌 것이다.
        * 왜 `Two Pointers`인가?
            * 배열에서 인덱스가 포인터 역할은 한다.
            * 현재 문제는 2개의 배열에 대해 각 원소를 서로 비교하고 있다. 일반적인 반복문과 조건식으로만 구현한다면 O(N^2) 시간복잡도가 될 수도 있다.
            * 포인터를 역방향으로 이동시키면 기존 배열에 대해 In-place 정렬을 손쉽게 구현할 수 있다.
        * 어떻게 했어야 `Two Pointers`를 생각할 수 있었을까?
            * 관리해야 하는 인덱스가 1개보다 많고 순회 조건이 단순 index++ 형식이 아니라면, `Two Pointers`로 풀 수 있는지 검토해야 한다.
* **🥸Pseudo code**
```java
// 아래 조건식에 기존 early exit이 포함되어 있기 때문에 별도 early exit 로직 불필요
// nums1의 마지막 원소 인덱스 변수 초기화
// nums1와 nums2에 대한 인덱스 'Pointer' 변수 p1, p2 초기화
    // 0부터 시작하는 게 아니라 역방향으로 인덱스 초기화
// while (p1과 p2 모두 0보다 클 때)
    // 조건식 분기 : nums1와 nums2의 요소를 비교해서 큰 값을 찾아 nums1 마지막 원소부터 역방향으로 할당한다
        // 후위연산자로 인덱스 값 변경
// while (p2가 0보다 클 때)
    // 역할 : nums2가 남을 경우 나머지 일괄 처리
    // p1에 대해서 나머지 일괄 처리할 필요 없는 이유 : 어차피 nums1에 대해 In-place 정렬하는데다가 이미 non-decreasing order로 정렬되어 있기 때문이다.
```

### ✅ [SUCCESS] 정답/모범답안 로직
* **✨ 수정된 흐름:**
```java
class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        // 각 배열의 유효한 맨 끝 원소를 가리키는 포인터
        int p1 = m - 1;
        int p2 = n - 1;
        // nums1 배열의 가장 마지막 빈 공간을 가리키는 포인터
        int p = m + n - 1;

        // 뒤에서부터 큰 값을 비교하며 채워 넣기
        while (p1 >= 0 && p2 >= 0) {
            if (nums1[p1] > nums2[p2]) {
                nums1[p--] = nums1[p1--];
            } else {
                nums1[p--] = nums2[p2--];
            }
        }

        // nums2에 원소가 남아있는 경우 일괄 복사
        // (nums1에 남은 원소는 이미 정렬된 제자리에 있으므로 이동할 필요 없음)
        while (p2 >= 0) {
            nums1[p--] = nums2[p2--];
        }
    }
}
```

### 🛠️ 내 코드에서 놓친 디테일 (Syntax / Edge Case)
* **💡 문법/라이브러리 피드백:**
    * 배열 원소 교체 방식 비효율적
        * `for-loop`보다 Java built-in method `System.arraycopy`을 사용했으면 더 좋았을 것이다
            * C/C++로 메모리 블럭을 복사하기 때문에 요소를 하나씩 순회하는 것보다 빠르고 효율적이다.
